## 敘事主線（一句話）

> 「我用一份真實的政府開放資料，做了一個讓任何人都能**自己探索**台北捷運如何隨城市與事件呼吸的互動網站。」

整場圍繞**三個問題**展開，demo 就是現場「回答」這三題：
1. 捷運運量怎麼隨時間與事件變化？（趨勢）
2. 哪些站最重要、資料怎麼才不會騙人？（排行 + 閘門陷阱）
3. 站點能反映城市的居住/就業結構嗎？（通勤潮汐）

## 逐段講法

### ① 開場鉤子 + 問題（0:45）
> 「大家每天都搭捷運，但很少人想過——這條線的運量其實藏著整座城市的行為訊號。疫情怎麼衝擊它？哪一站其實被低估了？哪些站是住宅區、哪些是辦公區？我做了一個網站，讓任何人都能自己點出答案。」

- 開場別念規格，先丟**共鳴 + 懸念**。
- 一句帶出後面三個問題。

### ② 資料來源（0:45）
> 「資料來自**政府開放資料平台 data.gov.tw 的臺北捷運公司**，兩份檔：一份是 **1998–2026 的逐月全系統運量與營收**，一份是 **1996–2025、134 站的年進出站人次**。月資料看趨勢，站資料看排行與通勤。都是官方真實資料，兩份年總量交叉比對約 97% 吻合。」

- 強調 **真實、官方、長時序、兩份互補**（對應評分「真實資料 20 分」）。
- **順手展示頂端「📥 下載原始開放資料」expander**：「而且資料完全開放——使用者可以**一鍵下載原始 CSV 自己核對**，不是只能看我做的圖。」（呼應開放資料 + 互動性，加分點）

### ③ Demo ①｜歷史趨勢 M1（1:30）
**黃金路徑（邊點邊說）：**
1. 「先看整體趨勢，指標選人次。」（指標 selectbox）
2. 「原始月資料抖動很大——」切到 **12 個月移動平均**「——換成移動平均，長期走勢就乾淨了。」
3. 「我在圖上標了**路網通車與政策事件線**：可以看到每條新線通車後的跳升，還有 2020 疫情這道斷崖。」（指事件線）
4. （可選）拖年份 slider 聚焦 2018–2022。

✓ 命中：多個 widget、標註/參考線。

### ④ Demo ②｜站點排行 M4（1:30）— 記憶點
1. 「再看站點排行，2025 年進出合計，台北車站第一。」
2. 「這裡有個關鍵：**合併同站不同閘門**。」**關掉** checkbox →「台北車站被拆成 R 和 BL 兩筆，排名瞬間掉到西門後面。」
3. **打開** →「合併後台北車站 5,545 萬，遠超西門 2,489 萬。**不處理資料就會得到錯結論**——這是整個專案我最在意的設計。」

✓ 命中：互動改變輸出、展現資料素養。這段是全場最強的「啊哈」時刻。

### ⑤ Demo ③｜AI 解讀（1:00）
1. 「每頁都有 **AI 解讀**，背後接 Claude API。」展開 expander。
2. 「它會把目前畫面的篩選與數據摘要傳給模型，產生白話解讀。」點**產生解讀**。
3. 等結果（約數秒），念出一兩句:「它正確抓到了台北車站的領先與合併的影響。」
4. 一句帶安全設計：「我**只傳算好的摘要**給它、不讓它自由生成數字，避免幻覺。」

⚠️ 前提：**Streamlit Cloud Secrets 的 `ANTHROPIC_API_KEY` 必須先填好**，否則只會顯示「尚未設定金鑰」。這是 demo 前唯一硬待辦。

### ⑥ 關鍵發現（1:00）— 講一個
**主打：通勤潮汐分群**
> 「最有趣的發現在通勤潮汐頁。我用『進站÷出站』的進出比把站分成兩群：比值 >1 是**住宅型**（居民早上從這進站去上班），<1 是**商業/轉乘型**。資料把市郊住宅站（新店、淡水）和市中心商辦/轉乘站（南港、板橋）分了開來——符合台北的居住與就業地理，是**純資料就能還原的城市結構**。」

**主動加一句但書（預防被反問「來回不是會抵消？」）：**
> 「要說明的是，純對稱來回其實會抵消，所以我量到的是**不對稱行程的淨傾向**——是淡淡的傾向、不是硬分類，多數站其實落在均衡帶。要看真正的尖峰方向得用逐時段 OD 資料，這正是我下一步想做的。」

> 備案（疫情）：「COVID 讓 2020 年 −11.9%、2021 −23.7%，是 SARS（−2.5%）近十倍的衝擊。」

### ⑦ 反思 + 收尾（1:00）
> 「最難的技術挑戰是**閘門合併**：站名後綴 R/BL/G 是路線碼，要正確去尾、合併同站，否則大轉乘站會被嚴重低估。我為這段寫了 pytest、用 GitHub Actions 自動跑，共 17 項測試。
>
> 下一步想加入**每日分時 OD 流量**做尖離峰分析，但全量約 30GB，得先做聚合管線——這是我課後想繼續的方向。
>
> 網址在這裡，歡迎大家現在就打開玩玩看。謝謝！」（貼網址到聊天室）

## 轉場句（讓段落不卡頓）

- 趨勢→排行：「看完整體，那**單一站**呢？」
- 排行→AI：「資料會說話，但**能不能自動幫人讀**？」
- AI→發現：「除了讀畫面，資料本身還藏了一個我覺得最有趣的東西。」
- 發現→反思：「要做到這些，最難的其實是……」

## 上台技巧

- **瀏覽器全螢幕**展示，不要露出 VS Code/終端機。
- 邊點邊**講出你在點什麼、為什麼**（「我把合併關掉，看會發生什麼」）。
- 語速放慢，每切一頁停半秒讓觀眾跟上。
- 眼睛看人，不要一直盯螢幕念。
- 出錯就**冷靜關掉分頁、重開網址繼續**——別道歉一長串。
