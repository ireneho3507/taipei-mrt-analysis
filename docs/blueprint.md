# 臺北捷運運量分析 — Streamlit App 藍圖

期末專案（NS5116 / 電腦硬體與程式語言在行為科學實驗與大數據分析之應用）
資料來源：data.gov.tw → 臺北大眾捷運股份有限公司

## 1. 資料來源（2 份，皆已驗證）

| 檔案 | 粒度 | 主要欄位 |
|------|------|---------|
| `臺北市捷運客運概況按月別.csv` | 月 × 全系統（1998-01 ~ 2026-03，連續無缺） | 客運人次(總/中運量/高運量)、平均每日客運人次、客運收入(總/中/高，千元) |
| `臺北市捷運各站進出站人次_年別.csv` | 年 × 站（1996 ~ 2025，30 年、134 站） | 進站人次、出站人次、進/出站增減率% |

> 註：「客運」即捷運載客；「中運量」= 文湖線、「高運量」= 其餘重運量路線。兩檔年度總量對得起來（≈97%）。

## 2. 分析模組（5 項）

### M1 歷史人次 & 營收趨勢（折線）
- 資料：客運概況（月）
- 精進：12 個月移動平均 / YoY 選項去鋸齒；雙 y 軸（人次＋營收）並可帶出**客單價 = 收入÷人次**；事件標註線（2008 文湖、2014 松山、2020 環狀、2020-21 疫情、2023 TPASS）；2026 半年截尾標示。
- Widgets：指標(人次/營收/客單價)、平滑方式(原始/12月MA/YoY)、年份範圍。

### M2 運量結構：中運量 vs 高運量（堆疊長條）
- 資料：客運概況（月→可彙年）
- 精進：**100% 堆疊**呈現比例；疊一條「文湖線佔比」趨勢（7.9%→11.7%→9.4%，疫情期上升）；可切換人次/收入（凸顯文湖線人次 9.5% vs 收入 10.6%）。
- Widgets：人次/收入、100%堆疊/絕對量、年份範圍。

### M3 季節與崩跌（淡旺季 + 疫情）
- 資料：客運概況（月）
- 精進：用 **`平均每日客運人次`** 去除月天數干擾；季節指數（各月÷當年均值）疊 28 年看月型態；年 YoY 標註 **SARS(2003 −2.5%)** 與 **COVID(2020 −11.9%、2021 −23.7%)**；加「回到疫情前高峰」復甦線；**年計算排除 2026**。
- Widgets：指標(月人次/平均每日)、季節指數開關、年份範圍。

### M4 站點人流排行
- 資料：各站進出站（年）
- 精進：**「合併同站不同閘門」toggle**（台北車站 R+BL 合併 5,545 萬 > 西門 2,489 萬，不合併會誤判榜首）；年份 slider + 名次變化(bump)；用 `增減率%` 另出「成長最快站」榜；過濾人次 0 站。
- Widgets：年份、排序基準(進/出/進+出)、合併閘門、Top-N。

### M5 站點通勤潮汐：早晚高峰方向流（每日分時 OD）
- 資料：每日分時各站 OD 流量（data.taipei），取 **2019-11、2025-11 兩代表月**，只計工作日，離線聚合成 `data/od_tidal.csv`（`scripts/od_aggregate.py`）。原始巨檔（每月約 270MB）不進版控。
- 內容：對每站分早高峰(07–09)、晚高峰(17–19)累計其作為「起站(出發)」與「訖站(抵達)」的人次。**潮汐指數 = 早晨出發佔比 − 傍晚出發佔比**：住宅站早出發·晚抵達(>0)、商業/轉乘站相反(<0)。主散點(早晨出發佔比 × 傍晚出發佔比，色＝潮汐指數、大小＝運量)＋疫情前後對照散點(2019 vs 2025)。
- 為何不用舊版「年進出比」：年資料下同站早晚來回會互相抵消，進出比僅 ±13%、訊號弱，住宅/商業推論無實證基礎。改用**分時方向 OD** 後，早晚為各自獨立時間窗，不會抵消，指數可達 ±0.7，且站名身分（蘆洲/新莊住宅、南港軟體園區/信義商業）完全對得上。
- Widgets：代表月份(2019/2025)、最少工作日均運量門檻。
- 依賴：純 pandas（聚合與分析皆不需 scikit-learn）。

## 5. 測試（pytest + GitHub Actions）

純函式測試（不需啟動 app 或 API）：
- `test_data_loader`：統計期年/月解析、2026 標為不完整年、閘門站名擷取/合併、編碼 utf-8-sig。
- `test_analysis`：年 YoY 排除 2026、排行「台北車站(合併) > 西門」、季節指數合理範圍、運量結構百分比加總=100%、客單價>0。

CI：`actions/setup-python` → `pip install -r requirements.txt` → `pytest`。
