# 🚇 臺北捷運運量分析 Streamlit App

NS5116《電腦硬體與程式語言在行為科學實驗與大數據分析之應用》期末專案
以臺灣政府開放資料製作的公開互動分析網站。

## 數據源

皆來自政府開放資料平台（[data.gov.tw](https://data.gov.tw)）／臺北市資料大平臺（[data.taipei](https://data.taipei)）／臺北大眾捷運股份有限公司：

| 檔案 | 粒度 | 內容 |
|------|------|------|
| `data/臺北市捷運客運概況按月別.csv` | 月 × 全系統（1998-01～2026-03） | 客運人次（總/中運量/高運量）、平均每日人次、客運收入 |
| `data/臺北市捷運各站進出站人次_年別.csv` | 年 × 站（1996～2025，134 站） | 各站進站/出站人次與增減率 |
| `data/od_tidal.csv` | 站 ×（2019-11、2025-11） | 各站早/晚高峰的出發·抵達與潮汐指數，供 M5 通勤潮汐使用 |

> `data/od_tidal.csv` 由「[每日分時各站 OD 流量](https://data.taipei/dataset/detail?id=63f31c7e-7fc3-418b-bd82-b95158755b4d)」原始檔（每月約 270MB）以 `scripts/od_aggregate.py` 離線聚合而成；原始巨檔不進版控。
> 「中運量」＝文湖線（膠輪系統），「高運量」＝其餘重運量路線。

## 功能

五個互動分析分頁：

| 分頁 | 內容 | 主要互動 |
|------|------|---------|
| 📈 歷史趨勢 | 人次／營收／客單價折線，含路網通車與疫情事件標註 | 指標、平滑（原始/12月移動平均/年增率）、年份範圍 |
| 🧩 運量結構 | 中運量 vs 高運量 100% 堆疊長條 | 人次/營收、百分比切換、年份範圍 |
| 🗓️ 季節與崩跌 | 季節指數（去天數干擾）＋年增率（標註 SARS／COVID） | 季節指標、年份範圍 |
| 🏆 站點排行 | 站點人流排行＋成長最快榜 | 年份、排序基準、**合併同站閘門**、Top-N |
| 🌗 通勤潮汐 | 早/晚高峰方向流散點（早晚出發佔比）＋潮汐指數，並做疫情前後（2019 vs 2025）對照 | 代表月份、最少運量門檻 |

## 執行方式（本機）

```bash
pip install -r requirements.txt
streamlit run app.py
# 開啟 http://localhost:8501
```

## 測試

純函式單元測試（不需網路）：

```bash
pip install -r requirements-dev.txt
pytest -q
```

GitHub Actions（`.github/workflows/ci.yml`）會在每次 push／PR 自動跑這些測試。

## 專案結構

```
app.py                    Streamlit 進入點（5 分頁）
src/
  data_loader.py          載入/清理（純 pandas）
  analysis.py             五模組分析純函式
  charts.py               Plotly 圖表 + 事件標註
tests/                    pytest（data_loader / analysis）
data/                     兩份開放資料 CSV + od_tidal.csv（OD 聚合小檔）
scripts/od_aggregate.py   把巨量分時 OD 原始檔離線聚合成 od_tidal.csv
.streamlit/config.toml    主題
.github/workflows/ci.yml  CI
docs/                     規則、藍圖、討論記錄
```

> `_od_raw/`（原始分時 OD 巨檔）已列入 `.gitignore`，不進版控；要重建 `data/od_tidal.csv` 時才需依 `scripts/od_aggregate.py` 註解下載對應月份。

## 限制

- **資料粒度不一**：站別資料僅到「年」，月趨勢與季節分析僅用全系統資料，無法做「各站逐月」分析。
- **2026 為不完整年**（僅 1～3 月）：年度分析已自動排除以免誤判崩跌。
- **分車種營收 2005–2007 缺漏**：開放資料該三年未提供中／高運量的收入拆分（人次正常），故運量結構頁切「營收」時該段為空白；本專案忠實呈現缺值，不自行補值。
- **轉乘站閘門**：站別（年）資料中同站不同路線會被拆成多筆（如台北車站 R／BL），排行分析提供「合併閘門」選項處理（OD 資料本身已用單一站名，無此問題）。
- **通勤潮汐取樣**：M5 採每日分時 OD 的**兩個代表月**（2019-11 疫情前、2025-11 近期），只計工作日。一個月已有約 20 個工作日可平均出穩定型態；受限於原始檔體積（每月約 270MB、全量約 30GB），未取全期間連續資料，故呈現的是「代表性月份」而非長期逐月演變。
- **潮汐指數的詮釋**：指數＝早高峰出發佔比 − 晚高峰出發佔比，住宅／商業為依此**方向反轉**的傾向描述，非行政區或土地使用的硬分類。
